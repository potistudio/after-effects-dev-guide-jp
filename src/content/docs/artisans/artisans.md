title: 'Artisan'
---
:::note
アーティザンの育成を検討されている方は、まずはご相談ください。

:::
Artisan API は、プラグインが 3D レイヤーのレンダリング出力を提供するために必要な関数フックを公開し、After Effects (2D レイヤーのすべてのレンダリングを引き続き処理します) から完全に引き継ぎます。 *コンポジション設定* > *詳細* ダイアログ内から選択できる、コンポジションごとに 1 人のアーティザンのみが存在できます。職人は 3D 環境をレンダリングし、After Effects にコンポジション内の各要素に関する情報を求めます。ご想像のとおり、これは膨大で退屈なプロセスです。この API は、After Effects の 3D レンダリングをオーバーライドする必要性が特にない人にはお勧めできません。

アーティザンは、通信するために書かれたエフェクトと情報を共有できますが、エフェクトがこの通信を開始することはできません。 Artisans が使用するスイートの多くは、すべてのエフェクトがレイヤーに適用された後にのみ生成されるレンダリング コンテキストを必要とします。

---

## インタラクティブな職人

これらは、コンポジション内のすべてのレイヤー (ユーザーが 3D 作成したレイヤーだけでなく) を処理する点で標準的なアーティザンとは異なり、オンスクリーン表示のためにのみ呼び出され、レンダリングされた最終出力に対して呼び出されることはありません (レンダリングはデフォルトのアーティザンに「フォールスルー」呼び出します)。
